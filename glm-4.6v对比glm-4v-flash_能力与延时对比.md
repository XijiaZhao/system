# glm-4.6v vs glm-4v-flash 视觉能力(情感 / 动物 / 人脸识别)与推理延时对比报告

> 测试对象:**glm-4.6v**(付费)与 **glm-4v-flash**(免费),智谱 bigmodel 兼容端点
> 统一条件:thinking 关闭、temperature 0.2、JSON mode、直连 API(不走项目 provider)、`stream=False`
> 测试日期:能力 2026-07-08-13(4.6v)/ 2026-07-14(flash);延时 2026-07-14(两模型)
> 数据来源:能力 `../results/`;延时 `results/latency_20260714_095227.csv`(4.6v)、`latency_20260714_102645.csv`(flash)
> 口径:两模型延时在不同时间窗采集,跨模型比较看量级与趋势;所有数据集均为受控库,数字为乐观上限。

---

## 一、总览一句话

- **能力**:两模型**粗粒度都强**(物种 100%、粗情感、同人召回 100%);**细粒度都弱且 flash 更弱**(品种 69%→43%,人脸 96%→84%)。
- **延时**:flash 全面快 **2.6~3.6 倍**,且**长尾几乎消失**(4.6v 的 p95 冲到 12132~19557ms(约 12~20s),flash 全收在 ~2500ms(约 2.5s))。
- **取舍**:flash 用"细粒度能力下降"换"低延时 + 稳定性",实时语音场景优势决定性;需要精细鉴别时用 4.6v。

## 二、识别能力对比

| 能力 | 数据集 | 指标 | glm-4.6v | glm-4v-flash | 差异 |
|------|--------|------|:---:|:---:|:---:|
| 情感(closed 严格) | CK+ | 8 类命中 | 49% | 49% | 持平 |
| 情感(open 归一后) | CK+ | 8 类命中 | 55% | 54% | 基本持平 |
| 动物-物种 | Oxford Pets | 猫/狗准确率 | **100%** | **100%** | 持平,零混淆 |
| 动物-品种 | Oxford Pets | 松匹配命中 | 69% | **43%** | flash ↓ 明显 |
| 人脸 1:1 总准确率 | CASIA-FaceV5 | 是否同一人 | 96% | **84%** | flash ↓ |
| └ 正样本召回 | | 同人认得出 | 100% | 100% | 持平 |
| └ 负样本拒识 | | 陌生人不误判 | 95% | **80%** | flash ↓ 更爱误认 |

**要点**:

1. **粗类判断两模型都稳**:物种 100% 零混淆;粗情感几乎不变;人脸"同人召回"都是 100%(不漏认自己人)。这是记忆模块真正要用的能力,两模型都扛得住。
2. **情感都两极分化**:高兴/平静/惊讶几乎全中,恐惧/轻蔑基本为 0 —— 模型能力边界,非问法问题。只适合当粗粒度氛围标签。
3. **细粒度 flash 更弱**:品种 69%→43%(flash 更多填"不确定",近似误认变多);人脸拒识 95%→80%,8 个错全是"陌生人→误当同人"(false accept),同人一个没漏 —— **flash 比 4.6v 更宽松**,区分"家人 vs 陌生来访者"误纳风险更高。

## 三、延时对比(单图延时专测,20图×3次,单位 ms)

**表头说明**(3.1 / 3.2 两表列头相同):

| 列 | 含义 |
|----|------|
| 图数 | 单次调用送入的图片数(情感/动物=1;人脸 1:1 验证=2) |
| 上传KB | 图片经 encode 归一成 RGB JPEG 后的平均字节(base64 前),即真正随请求上传的体积,非磁盘原始文件大小 |
| **pN(分位数)** | 全部调用延时从小到大排队,pN=第 N% 位置的值,即"N% 的请求比它快"。**p50**=中位数(典型体感);**p90/p95/p99**=越靠后越反映长尾最坏情况(p95 常用来定"绝大多数用户能接受的上限") |
| 均值 / 标准差 | 平均延时 / 波动幅度。标准差越接近均值,说明分布越长尾(少数请求拖得很久) |
| 均tok | 平均输出 token 数(completion_tokens);自回归生成下,输出越长延时越高 |
| ms/tok | 平均"每输出 token 毫秒数"=api 延时 / 输出 token。**注**:混了 prefill(处理图片+prompt)与 decode(生成),图多/prompt 长会抬高此值,不能单读成"生成慢" |

> 延时口径:`total = encode(本地读图编码,1.5~3.9ms 可忽略) + api(网络往返)`,`stream=False`,测整段生成完成的总延时(非首 token);顺序单发、已剔除预热/重试/失败调用。

| 情感 | 1 | 53.6 | 2470 | 9485 | 12132 | 14102 | 3976 | 3638 | 19.7 | 212 |
| 动物 | 1 | 55.6 | 2748 | 13584 | 19557 | 22616 | 5411 | 5700 | 44.4 | 122 |
| 人脸 | 2 | 102.2 | 3463 | 13966 | 15798 | 24842 | 5697 | 5830 | 22.6 | 257 |

|------|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 情感 | 1 | 53.6 | **724** | 1559 | 2459 | 2696 | 943 | 538 | 18.7 | 50.5 |

### 3.3 加速倍数(4.6v → flash)

| 任务 | p50 | p95 |
|------|:---:|:---:|
| 情感 | 2470→724(**3.4×快**) | 12132→2459(**4.9×快**) |
| 动物 | 2748→1074(**2.6×快**) | 19557→2677(**7.3×快**) |
| 人脸 | 3463→966(**3.6×快**) | 15798→2415(**6.5×快**) |

**两个关键变化**:

1. **典型延时快 3~4 倍**:flash 三任务 p50 全落到 724~1074ms(约 1s 上下),4.6v 是 2470~3463ms(约 2.5~3.5s)。
2. **长尾几乎消失(更重要)**:4.6v 标准差 ≈ 均值、p95 冲到 12132~19557ms(约 12~20s),同图重复能差 5~6 倍;flash 标准差 ~500ms(约 0.5s,仅均值一半)、p95 全收在 2415~2677ms(约 2.5s)。**flash 单图延时可预测,4.6v 极度长尾**。对实时语音场景,这比典型值更关键。

> 两模型共性:本地 encode 均可忽略(1.5~3.9ms),`total ≈ api`;人脸最慢因双图(上传量翻倍),动物 tok 最多(输出含 action)。

## 四、健康度与兼容性

| 项 | glm-4.6v | glm-4v-flash |
|----|:---:|:---:|
| **thinking(思考模式)** | **关闭** | **关闭** |
| JSON 合法率 | 100% | 100% |
| 调用错误 / 429 | 0 / 0 | 0 / 0 |
| max_tokens 上限 | 1500 | **1024**(超限报 1210) |
| 计费 | 付费(6M token 额度) | 免费 |

> flash 必须把 `vlm_client.py` 的 `MAX_TOKENS` 降到 ≤1024;各任务实际输出 <50 token,不截断、不影响可比性。其余参数两模型全兼容。

## 五、选型建议

| 场景 | 推荐 | 理由 |
|------|------|------|
| 实时语音 + 粗类判断(有猫狗、是不是刚才那人、粗情绪) | **glm-4v-flash** | 延时 ~1000ms(约 1s)级、长尾稳、免费;粗类能力持平 4.6v |
| 需要细粒度(具体品种、严格区分陌生人/熟人) | **glm-4.6v** | flash 品种 43%、人脸拒识仅 80% |
| 对误纳陌生人敏感(门禁式"家人 vs 访客") | glm-4.6v 或加二次确认 | flash false accept 偏高 |

**对记忆模块**:粗粒度氛围/物种/同人重识别优先用 flash(延时体验远好、免费);凡涉及"精细鉴别"或"拒识陌生人"的决策,用 4.6v 或加确认环节,别单独依赖 flash。

## 六、局限与复现

- **不同时间窗**:能力与延时分开采集,两模型延时也非同一时刻;跨模型延时比较看量级与趋势,不当精确 SLA。
- **单次采样、n≈60**:p95/p99 不稳,作量级参考;要当 SLA 需多时段各跑一轮、`REPEAT` 提到 5。
- **受控数据集 = 乐观上限**:CK+/CASIA/Oxford 均为规整正脸/清晰图,真机(自然光、非正脸、遮挡)能力会降、图更大延时会升。

**复现**:
```bash
conda activate zerone-server
cd newserver/main/xiaozhi-server/test/pipeline_tests/vision_eval
# 切模型:改 vlm_client.py 顶部 MODEL(glm-4.6v / glm-4v-flash);flash 需 MAX_TOKENS<=1024
python kill_switch.py         # 门禁:JSON 稳定性
python run_emotion.py         # 情感能力
python run_animals.py         # 动物能力(先 sample_fix.py 固化)
python run_face_verify.py     # 人脸能力
cd latency && python run_latency.py   # 三任务延时
```
