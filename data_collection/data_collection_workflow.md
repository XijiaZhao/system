# 数据采集流程说明（两轮采集总结）

## 一、采集目标
- 获取机器人唇部舵机指令与对应图像数据的配对数据集，用于后续模型训练。
- 两轮采集分别覆盖不同范围的舵机运动组合，逐步优化采集策略和数据质量。


## 二、硬件配置（两轮通用）

| 项目 | 配置 |
|------|------|
| 舵机控制板 | Hiwonder LSC-32 |
| 舵机编号 | 15~27（共13个唇部舵机） |
| 相机型号 | 海康 MV-CU060-10UC |
| 相机分辨率 | 采集时固定为640×480 |
| 串口波特率 | 9600 |
| 舵机运动时间 | 500ms |
| 归一化基准 | 180 作为 1.0 |


## 三、第一轮采集（facesetC）

### 采集脚本
`collect_traversal.py`

### 采集策略
- 采用固定点位全组合遍历，共 **38880 张**样本
- 分6组采集，每组6480张，约3.6小时/组
- 支持断点续采

### 舵机点位设计
- **配对舵机**：左右对称舵机组合（如ID24/26、ID21/22等）
- **独立舵机**：单舵机独立点位（如ID17、ID27、ID15等）
- 归一化值范围均基于180尺度映射到PWM

### 数据保存
- 原始图（uncropped）
- 480×480 全脸图
- 256×256 嘴唇图
- 舵机指令 JSON 记录


## 四、第二轮采集（facesetD）

### 采集脚本
`collect_facesetD.py`

### 采集策略
- 共 **10368 张**样本，一次性遍历完成
- 支持断点续采，每100张自动保存JSON

### 舵机点位设计
- **配对舵机**：采用手工配好的归一化点位对（左舵机值, 右舵机值），不再使用镜像公式计算
- **独立舵机**：点位列表直接使用
- 部分舵机范围在第一轮基础上进行了扩大

### 数据保存
- 仅保存 480×480 全脸图和 256×256 嘴唇图（不再保存原始大图）


## 五、两轮对比

| 对比项 | 第一轮 | 第二轮 |
|--------|--------|--------|
| 总样本数 | 38880 | 10368 |
| 采集方式 | 分组采集（6组） | 一次性遍历 |
| 舵机配对 | 镜像公式计算 | 手工配对点位对 |
| 保存内容 | 原始图+全脸+嘴唇 | 仅全脸+嘴唇 |
| 串口号 | COM7 | COM8 |
| 舵机等待时间 | 1.0s | 1.2s |


## 六、采集流程通用步骤

1. **硬件连接**：连接 LSC-32 舵机控制板和相机，确认串口和相机识别正常。
2. **运行脚本**：根据采集轮次选择对应的脚本，并指定参数（如 `--test` 测试模式）。
3. **监控采集**：观察终端输出，确认舵机运动和拍照正常。
4. **数据保存**：采集完成后，检查输出目录中的图像、JSON 和 NPZ 文件是否完整。
5. **断点续采**：若中断，再次运行脚本会自动从断点继续。


## 七、补充说明

- 第一轮采集的原始图（uncropped）占用空间较大，第二轮已移除该保存项。
- 两轮采集的归一化基准一致（180 作为 1.0），便于数据混合使用。
- 第二轮采集脚本新增了 `--reset` 参数，可用于单独归位舵机。

## 十一、常见问题与处理

### 11.1 串口连接失败

**现象**：脚本报 `[错误] 无法打开串口`

**可能原因**：
- 舵机控制板未上电
- 串口号与脚本中的不一致
- 上位机软件占用串口

**解决方法**：
1. 检查舵机控制板供电
2. 确认设备管理器中的串口号，修改脚本中的 `SERIAL_PORT`
3. 关闭上位机软件，释放串口

### 11.2 相机未找到

**现象**：脚本报 `[错误] 未找到海康相机`

**解决方法**：
1. 确认相机USB线连接正常
2. 打开海康MVS软件，确认相机能被识别
3. 检查SDK路径 `SDK_PATH` 是否正确指向MVS安装目录下的 `MvImport` 文件夹

### 11.3 采集中断后如何继续

- 脚本支持自动断点续采，直接再次运行即可从断点继续
- 第二轮采集通过检查 `face_480/` 和 `lip_256/` 目录中的文件编号来判断进度

### 11.4 采集速度慢

- 影响速度的主要参数是 `SERVO_WAIT_TIME`（舵机到位等待时间）
- 第一轮为1.0s，第二轮为1.2s
- 可根据实际舵机响应情况适当减小该值，但需确保舵机到位

### 11.5 输出文件缺失或损坏

- 第二轮采集每10张和每100张会自动保存进度JSON
- 若中断，可通过 `records_current.json` 恢复记录
- 采集完成后会自动打包 `images.npz` 和 `commands.npy`

## 十三、名词解释

| 术语 | 说明 |
|------|------|
| PWM | 脉冲宽度调制，用于控制舵机位置的信号值 |
| 归一化值 | 将舵机位置映射到0~1区间，便于算法处理 |
| 断点续采 | 采集中断后重新运行脚本，能从断点继续 |
| LSC-32 | Hiwonder 舵机控制板型号 |
| facesetC/D | 数据集命名，C为第一轮，D为第二轮 |


## 十四、补充说明

### 14.1 环境准备

在运行采集脚本之前，需要先准备好 Python 环境：

1. 创建并激活 conda 环境：
```bash
conda create -n lipenv python=3.9
conda activate lipenv
```

2. 安装所需依赖：
```bash
pip install pyserial numpy opencv-python
```
注意：采集脚本依赖海康工业相机的 MVS SDK，需要提前安装 MVS 并确保 SDK_PATH 指向正确的 MvImport 目录。

### 14.5 失败重试机制
第一轮采集：遇到抓帧失败会打印警告并跳过，继续采集下一张

第二轮采集：内置 MAX_CAPTURE_RETRIES = 2，抓帧失败会自动重试，重试仍失败则跳过

中断后重新运行脚本，会自动从断点继续
