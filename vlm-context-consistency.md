# VLM 场景描述一致性：上下文感知提示

## 背景

当前 VLM 场景描述链路（`_capture_facial_expression` → `call_camera_tool` → VLLM）每次调用完全无状态：VLM 不知道它上一次对同一场景说了什么。结果是同一个静态画面在连续几次独立调用中可能产出事实层面矛盾的描述（如"微笑"→"面无表情"），破坏两个下游：

- **LLM 对话上下文**：注入矛盾的视觉信息，智能体"看到的"前后不一致
- **心情状态机**（`mood-state-machine`）：基于 LLM 回复 emoji 驱动，间接受视觉上下文影响，噪声输入导致心情漂移

目标：语义一致性——核心事实不矛盾，措辞允许自然变化。

## 方案：缓解 A — "措辞参考"提示

### 思路

不改变 VLM 调用的无状态本质，但把上一次缓存描述以**"措辞参考"**身份追加到 VLM prompt 中，让模型独立看图的同时获得风格锚点。三重去锚指令防止 VLM 盲从旧描述：

1. "措辞参考"而非"上一轮描述"——降权为风格建议
2. "仅供参考，不影响内容判断"——明确免责
3. "必须基于当前画面独立判断"+"以当前画面为准"——正向强调独立看图

### 设计决策

- **参考信息追加在 prompt 末尾**：VLM 先读完任务指令（"请用一句简短的中文描述…"）建立"独立看图"的心智框架，最后才看到参考文本。避免参考信息出现在开头时 VLM 产生锚定效应。
- **`get_current_description` 不检查新鲜度**：旧描述即使已过期（超过 TTL），措辞风格仍有参考价值。是否过期由 `visual_context.store` 的写入方管理，读取方不应以此过滤。
- **不改变 `call_camera_tool` 签名**：`question` 本来就是透传给 VLM 的，这里只是在透传前做字符串拼接，零接口变更。
- **线程安全**：`conn.visual_context` 是单个不可变对象（`VisualContext` dataclass），CPython GIL 下属性读取天然原子。`get_current_description` 在事件循环线程调用（与写入方同线程），更无竞争。

## 错误处理与边界

- 首次拍照（无缓存描述）→ `get_current_description` 返回 None → 使用原 prompt，行为与当前完全一致
- 上一次拍照失败（缓存未更新）→ 同首次
- VLM 仍然漂移（忽略 prompt 指令）→ 超出本方案范围，后续可叠"图像门控"或"定期重置基线"（缓解 B/C）
