# 仿生机器人头 · 大脑系统设计文档

> 定位：`main`（xiaozhi-server）作为**认知层大脑**，与硬件仿生机器人头（舵机）配合，
> 构建一个越来越像"真人"的、可养成的交互实体。
>
> 本文给出**整体框架 + 功能清单 + 功能间交互 + 分阶段开发路径**。

---

## 一、整体架构：给"大脑"补上"神经"和"肌肉"


```
┌─────────────────────────────────────────────────────────┐
│  感知层 Perception   听(ASR) │ 看(VLM/摄像头) │ 传感器      │
├─────────────────────────────────────────────────────────┤
│  认知层 Cognition    LLM调度 │ 情感理解 │ 记忆(长/短)       │
│                      视觉记忆(长/短) │ 情绪状态机 │ 人格      │
├─────────────────────────────────────────────────────────┤
│  表现层 Expression   说(TTS) │ 唇形 │ 表情 │ 动作 │ 注视     │
├─────────────────────────────────────────────────────────┤
│     控制协议层  Robot-MCP  (set_expression / play_action /  │
│                            look_at / blink …)  → ESP32舵机  │
├─────────────────────────────────────────────────────────┤
│     事件总线 Event Bus + 行为仲裁 (谁能控制脸/嘴，谁优先)     │
└─────────────────────────────────────────────────────────┘
```

**两个最关键的新基础设施**（没有它们，后面的功能都做不干净）：

1. **Robot-MCP 执行器协议** —— 把"眨眼 / 表情 / 转头"统一成一组 MCP 工具，LLM 和其它模块都通过它控制硬件。
2. **事件总线 + 行为仲裁** —— 让"看到猫"能触发"说话 + 转头"，同时解决"对话动作"和"待机眨眼"抢舵机的冲突。

## 二、功能清单（按模块）

### A. 表现层：表情与动作
### 现阶段负责人：越康 文奇

| # | 功能 | 实现路径 / 技术 |
|---|------|----------------|
| A1 | **情感→表情映射** | LLM 已能出情感标签(happy/sad/angry/surprise…) → 映射到一组**表情预设**(眉/眼/嘴角舵机姿态) → 经 Robot-MCP 下发 |
| A2 | **动作原语库** | 定义 blink / nod / shake / tilt(歪头) / look_at 等，每个是一段**舵机关键帧序列**，封成 MCP function，LLM 可主动调用 |
| A3 | **待机微动作(Idle)** | 独立于对话的**本地行为循环**：随机眨眼、轻微头部呼吸晃动、眼球扫视(saccade)。让它"没说话时也活着" |
| A4 | **注视/视线跟随** | VLM/人脸检测给出方位 → look_at 转头转眼看向说话人或移动物体 |
| A5 | **情感强度与过渡** | 表情用插值(easing)平滑过渡、按语气给强度，不要突变；离散标签→情感向量(valence/arousal) |
| A6 | **唇形升级(viseme)** | 当前是音频包络驱动，可升级为音素→viseme 对齐(TTS 时间戳)，口型更准 |


### B. 控制协议层：Robot-MCP
### 现阶段负责人：子昂
| # | 功能 | 实现路径 |
|---|------|---------|
| B1 | **机器人执行器 MCP Server** | 复用 xiaozhi 现有 MCP 接入点，定义工具：`set_expression(emotion,intensity)`、`play_action(name)`、`look_at(x,y)`、`blink()`、`set_servo(id,angle)`。ESP32 端实现执行 |
| B2 | **语音-表情时间同步(A/V sync)** | 文本分句时打情感/动作标记，与对应音频段一起下发或用时间戳对齐——说到"惊讶"时脸同步惊讶 |

### C. 记忆系统（对话侧）
### 现阶段负责人：子昂， 思颖

| # | 功能 | 实现路径 |
|---|------|---------|
| C1 | **短期/工作记忆** | 当前会话上下文 + "今日待办/临时事项"结构化存储，带过期清除 |
| C2 | **长期记忆(人物画像)** | 从对话抽取事实 → 去重/更新 → 存入**结构化档案**(persona + 事实库 + 向量检索)。就是"CLAUDE.md 式"的生活习惯/偏好 |
| C3 | **固化/遗忘算法** | 每条记忆带 时间戳+出现次数+情感权重+最近访问；定时任务(cron)：**高频/高情感 → 固化为长期；长期未访问 → 归档遗忘**。养成感的核心 |
| C4 | **检索与注入** | 对话时向量+关键词检索相关长期记忆，注入 prompt |
| C5 | **联想→惊喜** | 新输入(对话或视觉)与长期记忆相似且久未提及 → 触发**惊喜行为**(surprise 表情 + 动作 + "这个跟你上次说的…一样！") |

### D. 视觉系统（VLM）
### 现阶段负责人：思颖

| # | 功能 | 实现路径 |
|---|------|---------|
| D1 | **视觉输入管线** | 摄像头定时拍(每2–5s) → VLM(qwen-vl等)/轻量检测(YOLO)+VLM → 结构化输出(人/物/场景/动作) |
| D2 | **短期视觉记忆** | 滑动窗口帧缓冲 + 事件检测(同一物体连续N帧出现) + 去抖 → 触发实时评论"有只猫走过来"，说完即忘 |
| D3 | **长期视觉记忆(频率)** | 每帧实体计数统计，超阈值(如出现10次)固化：熟悉环境、常见人物数、物品常放位置 |
| D4 | **人脸识别与人物绑定** | 人脸 embedding 聚类区分不同人 → 通过对话学到名字 → 绑定到 C2 人物档案("认得你") |
| D5 | **视觉→行为** | 看到人脸→转头注视+微笑；看到熟人→开心；看到新东西→好奇。连接 D→A |

### E. 主动性与人格 —— 让它"像真人 / 养成" - 可以融合在记忆里
### 现阶段负责人：思颖，子昂
| # | 功能 | 实现路径 |
|---|------|---------|
| E1 | **主动发起对话** | 事件驱动(看到人来/该提醒的事/该关心的事) → 判断"是否该开口" → 主动说 |
| E2 | **情绪状态机(mood)** | 持续的"心情"变量(valence/arousal)，受交互更新、随时间衰减回基线，影响所有回应和表情基调。被夸→开心一阵 |
| E3 | **好奇心/自主创造** | 空闲或触发时让 LLM(高温度)结合当前记忆/视觉情境自主发起：讲笑话、提问、发感想。即"GPT 创造力" |
| E4 | **关系/亲密度养成** | 记录与每个人的互动历史/亲密度，越熟态度越亲近随意 |
| E5 | **稳定人格 persona** | system prompt 固定性格 + 长期记忆动态部分(口头禅、喜好演化) |

### F. 基础设施
### 现阶段负责人：最后整合时再规划
| # | 功能 | 说明 |
|---|------|------|
| F1 | **事件总线** | 内部 pub/sub，感知/认知/表现解耦，让视觉事件能触发动作与说话 |
| F2 | **行为仲裁/优先级** | 多模块争夺脸和嘴时的抢占：对话 > 待机；惊喜 > 常规。避免动作打架 |
| F3 | **调度(cron)** | 记忆固化、视觉频率统计、主动提醒的定时任务 |

## 三、功能间的交互（"活人感"的真正来源）

"遇到相似长期记忆→惊喜"是典型的跨模块链路。核心几条：

```
情感提取 ──────────────► set_expression (A1)         [说话时的表情]
TTS音频 ───────────────► 唇形 (已有) + A/V同步(B2)
文本情感标记 ──────────► 表情/动作与语音时间对齐

VLM短期(D2) ──┬────────► 主动评论 TTS
              └────────► 注视/转头 look_at (A4/D5)

VLM每帧实体 ──► 频率统计 ──► 长期视觉记忆(D3) ──► 人物档案(C2/D4)
对话事实抽取 ──────────► 长期记忆固化(C2/C3)

【惊喜链路】新输入(对话或视觉) → 检索长期记忆(C4)
   → 相似且久未提 → 惊喜(surprise表情 A1 + 动作 A2 + 话语 TTS)

情绪状态机(E2) ──► 给所有表情/回应染上当前"心情"基调
────────── 全部经由 事件总线(F1) 流转，行为仲裁(F2) 决定谁先执行 ──────────
```
